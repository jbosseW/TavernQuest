KEY VARIABLES:
• enemyCardW       - Width of each enemy card (130→115→105→85)
• enemyCardH       - Height of each enemy card (155→135→120)
• enemyPortraitSize - Portrait size (70→60→50)
• enemiesPerRow    - How many enemies fit per row (dynamic)
• numRows          - Total rows needed (1, 2, or 3)
• enemyZoneH       - Total height for enemy zone (180→290→385)

LAYOUT ALGORITHM:
1. Count total enemies
2. Select tier based on count (LARGE/MEDIUM/COMPACT)
3. Calculate enemies per row (ceil(count / numRows))
4. For each enemy:
   - Calculate row index: floor(index / enemiesPerRow)
   - Calculate column index: index % enemiesPerRow
   - Calculate row-specific enemy count
   - Center row independently
   - Position card at (rowStartX + col * spacing, topY + row * spacing)

ENEMY COUNTER BADGE:
• Appears when numEnemies >= 5
• Shows alive/total count
• Position: top-right, 65w x 22h
• Colors: Red border, semi-transparent background
• Updates dynamically as enemies are defeated

FONT SCALING:
• Name labels:     11pt → 9pt for 9+ enemies
• HP bar text:     Standard for all (readable even small)
• FAINTED overlay: 14pt → 11pt for 9+ enemies
• Portrait icons:  Scale proportionally to portrait size

MINIMUM SIZE PROTECTION:
• Cards never shrink below 85px width
• Ensures text remains readable
• HP bars maintain minimum 14px height
• Borders remain 2-4px thick
